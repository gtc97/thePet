## 一、项目概述

### 1.1 产品定位

**thePet** 是一款以**宠物档案记录与展示**为核心，**上门喂养、在线沟通**为附加功能的轻量化多端应用。

### 1.2 目标用户

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| **宠物主** | 25-40岁，一二线城市，上班族 | 记录宠物成长、出差宠物照料、寻找可靠服务 |
| **上门师傅** | 热爱宠物，有空闲时间 | 灵活增收、服务标准化保障、安全接单 |

### 1.3 产品原则

- 严格区分首期开发与待定功能，直播功能仅预留
- 用户隐私数据默认私密，地址脱敏展示
- 普通用户与管理端账号完全隔离
- 核心数据防丢失、防误删
- 禁止复杂社交、排行榜、内容推荐、会员体系

### 1.4 项目结构

```
thePet/
├── pet-app/            # UniApp X 用户端（微信小程序 + Android + iOS）
├── pet-server/         # Express API 后端
├── pet-admin/          # Vue3 管理后台
└── docs/               # 项目文档
```

## 二、技术架构

### 2.1 技术栈

| 层级 | 技术 |
|------|------|
| 用户端 | UniApp X (Vue3 Composition API) |
| 后端 | Node.js + Express + TypeScript |
| 数据库 | MySQL + Prisma ORM |
| 缓存 | Redis |
| 管理端 | Vue3 + Vite + Element Plus |
| 实时通讯 | Socket.IO |

### 2.2 核心设计决策

| 决策 | 方案 | 原因 |
|------|------|------|
| 身份模型 | User.roles JSON数组 | 支持双身份共存 |
| 订单状态机 | 枚举 + 状态日志表 | 完整审计追踪 |
| 隐私控制 | Pet.privacy + PetShare字段级 | 双层控制，灵活可配 |
| 文件存储 | 本地 + Sharp压缩 | 开发阶段简单，OSS就绪 |
| 聊天限制 | 1:1绑定订单 + 时间窗口 | 安全防骚扰 |
| 押金管理 | per-user deposit + per-order freeze | 简化资金流转 |

## 四、产品完善方向

### 4.1 核心体验优化

**🔹 用户留存与活跃**

| 优化项 | 当前问题 | 优化方案 | 优先级 |
|--------|----------|----------|--------|
| 宠物档案入口深度 | 静态展示，互动性不足 | 增加成长里程碑提醒（生日、疫苗到期、驱虫提醒） | 高 |
| 内容消费闭环 | 分享广场缺少发现机制 | 增加「精选宠物」「热门日记」推荐 | 中 |
| 社交互动增强 | 收藏功能较单薄 | 增加宠物互动（虚拟喂食、装扮）提升情感连接 | 中 |

**🔹 服务履约体验**

| 优化项 | 当前问题 | 优化方案 | 优先级 |
|--------|----------|----------|--------|
| 实时定位追踪 | 用户无法查看师傅位置 | 增加服务轨迹可视化 | 高 |
| 服务标准化 | 服务项偏基础 | 增加增值服务（梳毛、陪玩、健康检查）套餐化 | 中 |
| 异常处理机制 | 订单取消缺少自动赔付规则 | 明确责任界定和自动赔付机制 | 高 |

### 4.2 商业闭环完善

## 五、推广价值评估

### 5.1 市场机会分析

**✅ 需求真实且高频**
- 养宠人群持续增长，宠物档案记录是刚性需求
- 上门喂养解决「出差/旅行宠物无人照料」痛点，场景明确

**✅ 竞争差异化**
- 市场上宠物APP多聚焦社交/电商，「档案+服务」一体化定位独特
- 隐私保护、风控体系是核心壁垒

**✅ 商业模式清晰**
- 服务佣金（抽成）
- 增值服务付费
- 会员订阅（高级档案功能）

### 5.2 推广策略建议

**🔹 冷启动阶段**
- 种子用户获取：宠物医院、宠物店合作，定向邀请养宠KOL
- 地推试点：选择1-2个城市（如成都、杭州）进行精细化运营

**🔹 增长策略**
- 社交裂变：分享宠物卡片得优惠券
- 私域运营：微信社群沉淀，定期举办「宠物成长故事」征集活动
- 异业合作：与宠物用品品牌、宠物保险联合推广

**🔹 留存策略**
- 打卡机制：连续记录日记获得徽章
- 服务提醒：宠物生日、疫苗到期等节点推送
- 会员体系：提供数据导出、专属客服等增值服务

### 5.3 风险评估

| 风险类型 | 风险描述 | 应对策略 |
|----------|----------|----------|
| 信任风险 | 用户担心陌生人上门安全 | 资质审核、押金担保、服务全程留痕 |
| 供给不足 | 初期师傅数量不足 | 新人补贴、培训体系、灵活结算 |
| 竞争风险 | 同类APP竞争 | 强化隐私保护、服务标准化差异 |

## 七、总结

### 产品优势
- 定位精准，聚焦宠物档案记录核心需求
- 隐私保护和风控体系完善，用户安全感强
- 技术架构清晰，扩展性良好

### 完善重点
- 优先补齐支付结算、地图服务等基础能力
- 优化供需匹配效率
- 建立数据驱动的运营体系

### 推广前景
- 冷启动可依托宠物场景线下资源
- 通过社交裂变和异业合作快速获取种子用户
- 商业模式清晰，增长路径明确
- 建议优先在1-2个城市做深做透，验证PMF后再逐步扩张
